Bom, pessoal,  agora a gente vai entrar em um capítulo muito importante em relação à arquitetura de solução para cloud. Você vai perceber que, nos dias de hoje, uma das coisas mais comuns é a pessoa subir uma solution architect para AWS, para Google, para Azure etc. Isso é muito comum. A maioria dos cargos, hoje em dia, sobre arquitetura de solução tem foco em uma determinada cloud. Porque grande parte do seu workload, das soluções que você vai criar, vai rodar na cloud. Então, não basta apenas entender os design patterns. Não basta apenas entender as soluções de forma geral. Você tem que entender sobre essas soluções e qual é o serviço correspondente que tem num determinado cloud provider. E isso é extremamente importante.

É bem comum que um arquiteto de solução tenha noções de cloud computing, porém, o ponto mais importante, quando a gente está falando sobre isso, é como eu consigo ter boas práticas para implementar a arquitetura de solução nas clouds. Porque uma coisa é você simplesmente falar como uma solução vai funcionar. Outra coisa é como vamos setar boas práticas para que a gente consiga rodar essa solução de uma forma mais eficiente, mais segura, que esteja compliance e que tenha compliance em relação a normas regulatórias. Então, não tem como fugir. Um arquiteto de solução tem que manjar o mínimo de uma cloud. E desenvolvedor também. 

Assim, não é porque você está codando, hoje em dia, que você não precisa ter a noção mínima de como funciona uma cloud. Inclusive, a gente vai ter módulo específico sobre cloud computing. Mas, neste momento, eu quero compartilhar um framework criado pela AWS (Amazon Web Services) como boas práticas para você arquitetar soluções. Para que essas soluções rodem da melhor forma possível. Então, nos próximos vídeos a gente vai entender os principais pilares desse framework da AWS. Vamos lá!
