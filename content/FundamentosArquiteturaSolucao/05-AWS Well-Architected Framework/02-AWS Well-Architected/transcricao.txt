Bom, pessoal, conforme eu falei no vídeo anterior, a gente tem que entender um pouco melhor como funciona e qual é o passo que você precisa dar para ser um arquiteto de solução, por exemplo, e conseguir entender as soluções de Cloud Computing. Mas independente de qualquer serviço Cloud que você for rodar em qualquer Cloud Provider, existem boas práticas que qualquer arquiteto de solução tem que entender. E o mais interessante, essas práticas também podem ser aplicadas mesmo que você esteja rodando toda a sua infraestrutura on-premise. Grande parte desses pilares serve para qualquer coisa que você for desenvolver ou arquitetar. 

E de onde vem toda essa história? Vamos lá. A AWS tem uma algo chamado de AWS Well-Architected. Ele é um framework. Quando a gente está falando de framework, não é um framework de código. Framework, basicamente, é uma base, é a forma como você vai agir, como você vai trabalhar no dia a dia. E a AWS criou um framework para uma boa arquitetura, o Well-Architected. E como funciona esse framework? Uma observação, esse framework não precisa ser usado necessariamente somente com a AWS. É isso que eu quero dizer. Você pode usar isso utilizando o OCI, o Azure, Google e etc. Inclusive os Cloud Providers têm documentos parecidos com esse da AWS Well-Architected. Então, como funciona esse framework? Ele tem diversas áreas e como essas áreas devem ser vistas. Então, vamos falar um pouco dessas áreas. Ele é um framework de boas práticas e as áreas são as seguintes: excelência operacional, segurança, confiabilidade, eficiência e performance e otimização de custos. Então, esses pontos têm a parte de sustentabilidade também. Essas são as áreas cobertas por esse framework e cada uma dessas áreas tem princípios. Cada parte dessas áreas tem pilares super importantes que você tem que levar em consideração. Qual vai ser o nosso jogo agora? A gente vai avaliar pilar por pilar e olhar os seus principais princípios. Mas a gente vai olhar esses princípios, porém, de uma forma ou de outra eu vou super recomendar que você leia esse framework inteiro. A documentação inclusive está embaixo desse slide. É só buscar no Google Well-Architected da AWS e você vai cair na página deste framework. Então, o maior ponto de importância é você entender os princípios de cada área e você conseguir seguir cada parte dessa.

 É muito difícil você seguir uma “parada” se você não tem ideia nenhuma de cloud, mas se você entender os conceitos, você começa a aplicar cada uma dessas áreas. É muito interessante porque acaba virando como se fosse uma espécie de uma checklist. O que você tem que saber são insights que vão fazer uma grande diferença no seu dia a dia. Então, vamos agora, a partir de agora, entender um pouquinho mais. Faremos um resumo de cada área e depois de entendermos cada área, a gente vai falar sobre os seus princípios.
